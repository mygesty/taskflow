# TaskFlow — 项目宪法

## 身份

TaskFlow 是一个轻量级团队任务协作平台，建立在严格的三层架构之上：前端 → BFF → 后端。

## 核心信念

1. **分层是墙，不是建议** — 跨层违规会破坏 CI，而不仅仅是违反约定
2. **边界处解析校验** — 所有外部输入在进入系统前必须用 Zod 校验
3. **共享工具优于手写辅助函数** — 类型、模式和工具的唯一事实来源
4. **测试证据胜于断言** — 特性只有在单元+集成+端到端测试全部通过后才算完成
5. **机械执行优于文档约束** — 如果一条规则重要，就把它编码到 Linter 或 CI 检查中
6. **渐进披露** — Agent 以 `AGENTS.md` 为地图开始，然后按需阅读更深层文档
7. **每次会话只做一个特性** — 专注防止上下文稀释和合并冲突
8. **干净交接** — 每次会话以已提交的代码、已更新的进度和清晰的状态结束

## 技术价值观

- **无聊胜过新颖** — 优先选择稳定、文档完善的技术
- **可组合性胜过巧妙性** — 小的可组合函数优于大型巧妙抽象
- **显式胜过隐式** — 命名导出、类型化参数、无魔法字符串
- **BFF 绝不接触数据库** — 它只编排后端 API 调用
- **前端绝不直接调用后端** — 始终通过 BFF 中转

## 反目标

- 这不是一个实时协作工具（暂且不是）
- 这不是一个企业级项目管理套件
- 这不是一个供他人扩展的框架或库
