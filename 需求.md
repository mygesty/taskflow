# TaskFlow — 产品需求文档

## 1. 产品概述

### 1.1 产品定位

TaskFlow 是一款面向 5~20 人小型团队的轻量级任务协作平台。通过看板视图将任务从创建到完成的全流程可视化，让团队成员对工作进展一目了然。

### 1.2 目标用户

| 角色 | 特征 | 核心诉求 |
|------|------|----------|
| 团队负责人 | 管理多个项目，关注整体进度 | 看板总览、任务分配、进度统计 |
| 团队成员 | 负责具体任务执行 | 接收任务、更新状态、协作沟通 |
| 新成员 | 刚加入团队，需要快速上手 | 清晰的任务列表、@提及引导 |

### 1.3 核心价值

1. **看板驱动** — 拖拽式任务流转，替代传统列表式管理
2. **轻量协作** — 无冗余功能，专注任务管理与团队沟通
3. **即时反馈** — 通知、@提及确保信息不遗漏
4. **数据洞察** — 任务统计与燃尽图辅助决策

## 2. 功能需求

### 2.1 用户认证（feat-001）

#### 用户故事

> 作为团队协作者，我希望注册账号并安全登录，以便在平台上管理我的任务和工作区。

#### 功能详述

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 邮箱注册 | 使用邮箱+密码注册，密码至少8位，含大小写与数字 | P0 |
| 邮箱登录 | 邮箱+密码登录，返回JWT双Token | P0 |
| Token刷新 | Access Token 15min过期，使用Refresh Token自动续期 | P0 |
| 登出 | 清除Token Cookie，前端清除本地状态 | P0 |
| 密码强度校验 | 前端实时提示密码强度，后端Zod校验 | P1 |
| 重复邮箱拒绝 | 注册时邮箱已存在返回409 CONFLICT | P0 |

#### 安全要求

- 密码使用 bcrypt (cost=12) 哈希存储
- Access Token 存于 httpOnly + Secure + SameSite=Strict Cookie
- Refresh Token 存于 httpOnly Cookie，7天有效期
- 登录接口限流：每IP每分钟5次

#### 验收标准

- [ ] 注册成功后自动登录并跳转Dashboard
- [ ] 登录失败返回清晰的错误提示（邮箱不存在/密码错误）
- [ ] Token过期后自动刷新，用户无感知
- [ ] 登出后Token Cookie被清除
- [ ] 并发注册同一邮箱只有一个成功

### 2.5 任务拖拽移动（feat-005）

#### 用户故事

> 作为团队成员，我希望通过拖拽在列间移动任务，以便直观更新任务状态。

#### 功能详述

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 列内拖拽排序 | 在同一列内拖拽调整任务顺序 | P0 |
| 跨列拖拽 | 将任务从一列拖至另一列 | P0 |
| 位置计算 | 拖拽放置时自动计算新position值 | P0 |
| 乐观更新 | 前端先更新UI再发请求，失败时回滚 | P0 |
| 移动通知 | 任务移动后通知被指派人 | P1 |

#### 技术实现

使用 `@dnd-kit/core` + `@dnd-kit/sortable`：
1. `onDragEnd` 计算目标列和目标位置
2. 乐观更新：Zustand store 立即移动任务
3. BFF调用 `PATCH /api/v1/tasks/:id/move`
4. 失败时：TanStack Query invalidate 回滚到服务端状态

#### 请求格式

```json
PATCH /api/v1/tasks/:id/move
{
  "targetColumnId": "col-uuid",
  "position": 2
}
```

#### 响应格式

```json
{
  "success": true,
  "data": {
    "id": "task-uuid",
    "columnId": "col-uuid",
    "position": 2,
    "previousColumnId": "old-col-uuid"
  }
}
```

#### 业务规则

- 拖拽仅限同一看板内
- 移动后原列和目标列的其他任务position自动重排
- 同一任务1秒内不允许重复移动（防抖）

#### 验收标准

- [ ] 拖拽到新列后任务出现在目标位置
- [ ] 刷新页面后位置保持
- [ ] 拖拽过程中其他任务位置不跳动
- [ ] 网络异常时拖拽自动回滚

### 2.6 评论与@提及（feat-006）

#### 用户故事

> 作为团队成员，我希望在任务下发表评论并@提及同事，以便围绕具体任务进行讨论。

#### 功能详述

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 发表评论 | 在任务详情中输入评论内容(必填,≤2000字) | P0 |
| 评论列表 | 按时间升序显示评论，包含作者头像、名称、时间 | P0 |
| @提及 | 输入@触发成员下拉，选中后插入@name | P0 |
| 提及通知 | 被@提及的成员收到COMMENT_MENTION通知 | P0 |
| 删除评论 | 仅评论作者可删除自己的评论 | P2 |
| 编辑评论 | 评论发出后5分钟内可编辑 | P2 |

#### @提及交互流程

```
输入 @ → 弹出工作区成员下拉列表
         ↓ 输入文字过滤
     选择成员 → 插入 @UserName 到文本
         ↓ 提交评论
     解析 @提及 → 为每个被提及者创建通知
```

#### 评论数据结构

```typescript
interface CommentResponse {
  id: string;
  taskId: string;
  author: {
    id: string;
    name: string;
    avatarUrl: string | null;
  };
  content: string;
  mentions: Array<{
    userId: string;
    name: string;
  }>;
  createdAt: string;
}
```

#### 业务规则

- 评论内容中 `@` 后跟工作区成员名视为提及
- 提及解析仅限当前工作区成员
- 每条评论最多提及10人
- 被提及者收到通知，通知内容包含评论摘要和任务链接

#### 验收标准

- [ ] 评论发表后立即出现在列表底部
- [ ] @下拉仅显示当前工作区成员
- [ ] 被@提及者收到通知
- [ ] 评论按时间升序排列
- [ ] 他人无法删除非本人的评论

### 2.7 通知系统（feat-007）

#### 用户故事

> 作为团队成员，我希望收到任务相关通知，以便及时了解工作动态。

#### 功能详述

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 通知铃铛 | 顶部导航栏显示通知图标+未读数Badge | P0 |
| 通知下拉 | 点击铃铛展开最近20条通知 | P0 |
| 标记已读 | 点击单条通知标记为已读 | P0 |
| 全部已读 | 一键标记所有通知为已读 | P1 |
| 通知跳转 | 点击通知跳转到关联任务详情 | P0 |
| 通知列表页 | 独立页面展示所有通知（分页） | P1 |

#### 通知类型与触发条件

| 类型 | 触发条件 | 通知标题模板 |
|------|----------|-------------|
| TASK_ASSIGNED | 任务被指派给用户 | "{指派人}将任务「{任务名}」指派给你" |
| TASK_MOVED | 被指派的任务被移至新列 | "任务「{任务名}」已移至{列名}" |
| TASK_DUE_SOON | 截止日期前24h | "任务「{任务名}」将在24小时内到期" |
| COMMENT_MENTION | 在评论中被@提及 | "{评论人}在「{任务名}」中提及了你" |
| MEMBER_INVITED | 被邀请加入工作区 | "{邀请人}邀请你加入工作区「{工作区名}」" |

#### 通知数据结构

```typescript
interface NotificationResponse {
  id: string;
  type: NotificationType;
  title: string;
  content: string | null;
  read: boolean;
  task: {
    id: string;
    title: string;
  } | null;
  createdAt: string;
}
```

#### 业务规则

- 通知按创建时间降序排列
- 仅产生与当前用户相关的通知（不通知操作发起者自身）
- 已读通知保留30天后自动清理
- 未读数上限显示99+（实际数可能更多）

#### 验收标准

- [ ] 收到通知时铃铛Badge数字更新
- [ ] 点击通知后Badge数字减1
- [ ] 全部已读后Badge消失
- [ ] 点击通知跳转到正确的任务详情
- [ ] 通知列表支持分页加载
