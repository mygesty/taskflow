# 运行测试

按以下策略执行测试：

1. **确定范围**：
   - 如果在修改特定文件 → 仅运行该测试文件
   - 如果在修改某个领域 → 运行该领域的测试
   - 如果跨领域修改 → 运行完整测试套件
2. **运行命令**：
   - 单元测试：`pnpm test:unit -- <路径模式>`
   - 集成测试：`pnpm test:integration -- <路径模式>`
   - 全部：`pnpm test`
3. **重试逻辑**（最多 3 次）：
   - 测试失败后重试一次
   - 再次失败则检查错误并修复代码（不是测试）
   - 第 3 次失败则停止，在 `issues.json` 中记录
4. **绝不要**：
   - 通过删除或跳过测试来使其通过
   - 在不理解原因的情况下修改测试期望值
   - 对失败测试使用 `.skip()` 或 `.todo()` 而不记录原因

报告：运行了哪些测试、通过/失败数量、检测到的不稳定测试。
