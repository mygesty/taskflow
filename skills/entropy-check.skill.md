# 熵值检查

扫描代码库中的衰变与漂移：

1. **死代码**：查找未使用的导出、不可达的文件、被注释的代码块
2. **过期文档**：比较 `docs/` 内容与实际代码行为
3. **失效引用**：查找指向不存在文件的导入
4. **质量记分卡**：更新 `docs/QUALITY_SCORE.md` 评级
5. **层级违规**：检查跨层边界的向上导入
6. **重复逻辑**：查找应使用共享工具的相似函数
7. **缺失测试**：查找没有对应测试文件的 service/repository 文件
8. **类型安全**：扫描 `any`、`@ts-ignore`、`as unknown`

对每个发现：
- 在 `issues.json` 中记录，标注严重程度（低/中/高）
- 高严重程度的直接创建修复 PR
- 添加到 `docs/decisions/tech-debt.md` 技术债务追踪器

报告：按类别汇总发现项，列出前 3 优先级。
